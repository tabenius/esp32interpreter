# Multi-Translator (ESP32)

Multi-Translator is an ESP32-based firmware project that captures speech from an I2S microphone (or Bluetooth), transcribes it (STT), translates it to one or many target languages, synthesizes speech (TTS) and plays audio through an I2S DAC or Bluetooth. The repository contains a complete Arduino IDE–compatible project with mocked STT/Translation/TTS modules and a Web UI for entering API keys and changing runtime settings.

## Features
- I2S microphone input (e.g., INMP441)
- I2S DAC output for 3.5 mm line/headphone out
- Bluetooth audio support (stubs; A2DP integration left as TODO)
- OLED (SSD1306) display and simple rotary encoder menu
- Persistent settings and API keys in Preferences
- Soft-AP Web UI for entering OpenAI / Claude (Anthropic) API keys and runtime settings
- Modular code: replace mocked modules with real API calls or on-device models

## Quickstart (Arduino IDE)
1. Install Arduino IDE (latest stable) and Board support for ESP32 (Espressif) using the Boards Manager.
2. Install libraries via Library Manager:
   - ArduinoJson (ArduinoJson by Benoit Blanchon) version 6.x
   - Adafruit SSD1306 and Adafruit GFX (for OLED)
   - The sketches use core ESP32 libraries (WiFi, WebServer, Preferences, Wire, driver/i2s) that come with the ESP32 Arduino core.
3. Open the folder containing `main.ino` in Arduino IDE (Sketch > Open).
4. Select the correct ESP32 board (e.g., "ESP32 Dev Module") and upload speed.
5. Build and upload to your ESP32 device.

## Hardware wiring (recommended defaults)
- I2S mic (e.g., INMP441): BCLK -> pin 14, LRCLK -> pin 15, DATA -> pin 32
- I2S DAC output: BCLK -> pin 26, LRCLK -> pin 25, DATA -> pin 33
- OLED (SSD1306) I2C: SDA -> pin 21, SCL -> pin 22
- Rotary encoder: A -> 34, B -> 35, Button -> 39
- Status LED -> pin 2

Adjust pins in `config.h` if your wiring differs.

## Web UI (enter API keys and settings)
After boot the device starts a Soft-AP with SSID `MultiTrans-XXXX` (XXXX derived from chip ID).
- Connect a phone or laptop to the AP (no passphrase) and browse to `http://192.168.4.1`.
- The Web UI lets you enter OpenAI and Claude API keys, choose input/output languages, routing mode, and other runtime options.
- Keys are stored in Preferences under namespace `multi_translator_keys` on the device flash.

Security note: The AP is unsecured and the Web UI uses HTTP only. Do not enter sensitive keys on untrusted networks or leave the device exposed.

## Where to integrate real APIs
- STT: `stt_module.cpp` provides `runSTT(AudioBuffer&, Language)` which is mocked; replace its body with a call to a cloud or on-device STT. Helpers exist in `ai_api_clients.h/.cpp` as examples for OpenAI/Anthropic.
- Translation: `translation_module.cpp` exposes `translateText()` and `translateMany()` — swap the mock with HTTP calls to your translation provider.
- TTS: `tts_module.cpp` exposes `synthesizeSpeech()`; replace the mock audio generator with API-produced WAV/PCM and fill `AudioBuffer`.
- Helper client examples: `ai_api_clients.h` / `ai_api_clients.cpp` (optional) demonstrate how to send WAV to OpenAI Whisper, use chat completions for translation, or request TTS. These are examples and require ArduinoJson, WiFiClientSecure/HTTPClient.

- `web_ui.*` — Soft-AP web interface for configuration

## Usage notes
- The project is complete and should compile in Arduino IDE after installing the listed libraries and ESP32 board support.
- The core speech pipeline is: audio input -> STT -> routing -> translation -> TTS -> playback -> history.
- STT/Translation/TTS are mocked for offline operation; the Web UI allows entering API keys to integrate cloud providers later.

## Troubleshooting
- Missing library errors: install via Library Manager and restart Arduino IDE.
- No Web UI: check Serial output for AP startup messages; ensure the ESP32 boots and `WiFi.softAP()` succeeds.
- Low audio quality: verify I2S pin wiring and sample rate. Use `config.h` to change pins and constants.
